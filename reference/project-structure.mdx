### Component Categories

<Tabs>
  <Tab title="Feature Components">
    **Feature Components** implement core application functionality:
    
    - `transaction-form/` - Create and edit financial transactions with reactive forms, category filtering, and date selection
    - `category-form/` - Manage income and expense categories with CRUD operations
    - `transaction-reports/` - Display transaction history with filtering and sorting capabilities
    - `report-chart/` - Visualize financial data using Highcharts integration
    - `login-form/` - User authentication with JWT token management
  </Tab>
  
  <Tab title="Layout Components">
    **Layout Components** provide application structure:
    
    - `layout/` - Main application shell that wraps authenticated pages, includes navbar and toast notifications
    - `navbar/` - Application navigation bar with routing links
  </Tab>
  
  <Tab title="UI Components">
    **UI Components** are reusable interface elements:
    
    - `ui-dropdown/` - Generic dropdown component with customizable options and formatting
    - `toast/` - Notification component for user feedback
    - `confirmation-dialog/` - Modal dialog for user confirmations
  </Tab>
</Tabs>

### Service Responsibilities

<AccordionGroup>
  <Accordion title="AuthService">
    **Location:** `services/Auth-service.ts`
    
    Manages user authentication and authorization:
    - JWT token storage and retrieval
    - Login endpoint communication
    - Token expiration validation
    - Logout functionality
    
    Key methods:
    - `login(username, password)` - Authenticates user and stores token
    - `isLoggedIn()` - Validates token and expiration
    - `getToken()` - Retrieves stored JWT token
    - `logout()` - Clears authentication state
  </Accordion>
  
  <Accordion title="TransactionService">
    **Location:** `services/transaction-service.ts`
    
    Handles all transaction-related operations:
    - Create, read, update, delete transactions
    - Monthly report generation
    - HTTP client integration with RxJS observables
    
    Key methods:
    - `createTransaction(transaction)` - Creates new transaction
    - `getAllTransactions()` - Retrieves all user transactions
    - `getTransactionById(id)` - Fetches single transaction
    - `updateTransaction(id, transaction)` - Updates existing transaction
    - `deleteTransaction(id)` - Removes transaction
    - `getMonthlyReport(type, year)` - Generates income/expense reports
  </Accordion>
  
  <Accordion title="CategoryService">
    **Location:** `services/category-service.ts`
    
    Manages transaction categories:
    - Category CRUD operations
    - Type-based filtering (Income/Expense)
    - API integration with typed responses
    
    Key methods:
    - `createCategory(category)` - Creates new category
    - `getAllCategories()` - Retrieves all categories
    - `getCategoryById(id)` - Fetches single category
    - `getCategoriesByType(type)` - Filters by Income or Expense
    - `updateCategory(id, category)` - Updates existing category
    - `deleteCategory(id)` - Removes category
  </Accordion>
  
  <Accordion title="ToastService">
    **Location:** `services/toast-service.ts`
    
    Provides user notifications:
    - Success/error message display
    - Centralized notification management
    - Integration with Toast component
  </Accordion>
</AccordionGroup>

## Guards Directory

Route protection implementation:

```
guards/
в”њв”Ђв”Ђ auth-guard.ts          # Functional route guard
в””в”Ђв”Ђ auth-guard.spec.ts     # Guard unit tests
```

The `authGuard` is a functional guard (`CanActivateFn`) that:
- Checks authentication status before route activation
- Redirects unauthenticated users to `/login`
- Uses dependency injection to access `AuthService`

See `guards/auth-guard.ts:5-15` for implementation.

## Interceptors Directory

```
interceptors/
в”њв”Ђв”Ђ auth-interceptor.ts       # JWT token injection
в””в”Ђв”Ђ auth-interceptor.spec.ts  # Interceptor tests
```

The `authInterceptor` is a functional interceptor (`HttpInterceptorFn`) that:
- Automatically adds Authorization header with JWT token
- Skips token injection for `/login` endpoint
- Clones requests to add authentication headers

See `interceptors/auth-interceptor.ts:5-23` for implementation.

## Testing Files

Test files follow the `.spec.ts` naming convention and are co-located with their source files:

- `*.spec.ts` - Unit tests for components, services, guards, etc.
- Vitest is configured as the test runner
- Tests use Angular testing utilities

## Best Practices

<Warning>
**File Naming Convention:** All TypeScript files use lowercase with hyphens (kebab-case), except for model files which may use PascalCase for clarity (e.g., `Auth.model.ts`).
</Warning>

<Tip>
**Co-location:** Related files (component, template, styles, tests) are kept in the same directory for better maintainability and discoverability.
</Tip>
