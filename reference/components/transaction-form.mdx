## Overview

The `TransactionForm` component provides a reactive form interface for creating financial transactions. It supports both income and expense types with dynamic category selection, date picking, and comprehensive form validation.

**Location**: `~/workspace/source/src/app/components/transaction-form/transaction-form.ts:19`

## Form Structure

The transaction form consists of five main controls:

```typescript
transactionForm = new FormGroup({
  date: new FormControl(null, [
    Validators.required,
  ]),
  amount: new FormControl(null, {
    validators: [Validators.required, Validators.min(1)]
  }),
  type: new FormControl('', [
    Validators.required,
  ]),
  description: new FormControl(null),
  category: new FormControl({value: null, disabled: true }, {
    validators: [Validators.required, Validators.min(1)],
  }),
});
```

### Form Controls

| Control | Type | Validators | Initial State | Description |
|---------|------|------------|---------------|-------------|
| `date` | `FormControl` | `required` | `null` | Transaction date (uses Flatpickr directive) |
| `amount` | `FormControl` | `required`, `min(1)` | `null` | Transaction amount |
| `type` | `FormControl` | `required` | `''` | Transaction type (Income/Expense) |
| `description` | `FormControl` | None | `null` | Optional transaction description |
| `category` | `FormControl` | `required`, `min(1)` | `null` (disabled) | Category ID |

## Lifecycle Hooks

### ngOnInit

Initializes the form and sets up reactive category loading:

```typescript
ngOnInit() {
  this.category.disable();

  this.type.valueChanges.subscribe(type => {
    if (!type) return;
    
    this.category.reset();
    this.category.enable();

    this.categoryService.getCategoriesByType(type).subscribe(response => {
      this.categories.set(response.data);
    })
  });
}
```

**Behavior**:
1. Disables the category dropdown initially
2. Listens for type changes
3. When type is selected, enables category dropdown
4. Fetches and updates categories based on selected type

### onSubmit

Handles form submission and transaction creation:

```typescript
onSubmit() {
  if (this.transactionForm.invalid) return;

  const transaction: createTransactionModel = {
    date: this.date.value ?? undefined,
    amount: this.amount.value!,
    type: this.type.value as 'Income' | 'Expense',
    description: this.description.value ?? undefined,
    categoryId: this.category.value!,
  }

  this.transactionService.createTransaction(transaction).subscribe({
    next: (response) => {
      this.transactionForm.reset();
      this.toastService.success('Transaction created successfully');
      console.log('Transaction created successfully:', response);
    }
  });
}
```

**Behavior**:
1. Validates form before submission
2. Creates transaction model from form values
3. Calls transaction service to create transaction
4. On success: resets form and shows success toast
5. Logs response to console
