## Filtering Methods

### filterByDate

Filters transactions based on selected date range:

```typescript
filterByDate(): void {
  if (!this.startDate && !this.endDate) {
    this.transactions$ = this.transactionService.getAllTransactions().pipe(map(res => res.data));
    return;
  }

  const start = this.startDate ? new Date(this.startDate) : new Date('1900-01-01');
  const end = this.endDate ? new Date(this.endDate) : new Date('2100-12-31');

  const filtered = this.allTransactions.filter(transaction => {
    const transactionDate = new Date(transaction.date);
    return transactionDate >= start && transactionDate <= end;
  });

  this.transactions$ = new Observable(observer => {
    observer.next(filtered);
    observer.complete();
  });
}
```

**Behavior**:
1. If no dates selected, shows all transactions
2. Uses default dates if only one boundary is set:
   - Start: `1900-01-01` (if not specified)
   - End: `2100-12-31` (if not specified)
3. Filters cached transactions client-side
4. Creates new observable with filtered results

**Date Range Logic**:
- Inclusive of both start and end dates
- Supports partial ranges (only start or only end date)

### clearFilters

Resets all filters and reloads all transactions:

```typescript
clearFilters(): void {
  this.startDate = '';
  this.endDate = '';
  this.transactions$ = this.transactionService.getAllTransactions().pipe(map(res => res.data));
```

**Behavior**:
1. Clears date filter inputs
2. Reloads all transactions from service

## Report Generation Logic

The component generates reports through:

1. **Initial Load**: Fetches all transactions on initialization
2. **Date Filtering**: Client-side filtering based on date range
3. **Dynamic Updates**: Observable pattern ensures template stays in sync

## Data Fetching Strategy

- **Initial Load**: Server fetch via `getAllTransactions()`
- **Filtering**: Client-side filtering of cached data
- **Filter Reset**: Re-fetches from server

**Performance Consideration**: All transactions are loaded once and filtered client-side. For large datasets, consider server-side filtering.
