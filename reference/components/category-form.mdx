## Overview

The `CategoryForm` component provides a simple reactive form interface for creating transaction categories. It allows users to define categories with a name and type (Income or Expense) for organizing financial transactions.

## Form Structure

The category form is a simple two-field reactive form:

```typescript
categoryForm = new FormGroup({
  name: new FormControl('', [Validators.required]),
  type: new FormControl('', [Validators.required])
});
```

### Form Controls

| Control | Type | Validators | Initial State | Description |
|---------|------|------------|---------------|-------------|
| `name` | `FormControl<string>` | `required` | `''` | Category name |
| `type` | `FormControl<string>` | `required` | `''` | Category type (Income/Expense) |

**Usage**: These getters simplify access to form controls in both the component class and template.

## Methods

### setType

Programmatically sets the category type:

```typescript
setType(value: 'Income' | 'Expense') {
  this.type?.setValue(value);
  this.type?.markAsTouched();
}
```

**Parameters**:
- `value`: Either `'Income'` or `'Expense'`

**Behavior**:
- Sets the type control value
- Marks the control as touched to trigger validation display

**Use Case**: Typically called from template buttons or UI interactions to set the transaction type.

### onSubmit

Handles form submission and category creation:

```typescript
onSubmit() {
  if (this.categoryForm.invalid) return;
  const categoryData: CategoryRequest = {
    name: this.name.value,
    type: this.type.value as 'Income' | 'Expense'
  }

  this.categoryService.createCategory(categoryData).subscribe({
    next: (response) => {
      this.categoryForm.reset();
      console.log('Category created successfully:', response);
    }
  });
}
```

**Behavior**:
1. Validates the form before proceeding
2. Creates a `CategoryRequest` object from form values
3. Calls the `CategoryService.createCategory()` method
4. On success:
   - Resets the form to initial state
   - Logs success message to console

**Validation**: Returns early if form is invalid, preventing submission.

## CRUD Operations

### Create

The component handles category creation through the `onSubmit()` method:

```typescript
this.categoryService.createCategory(categoryData).subscribe({
  next: (response) => {
    this.categoryForm.reset();
    console.log('Category created successfully:', response);
  }
});
```

**Note**: Update, delete, and read operations are not implemented in this component. This is a create-only form.

## Type Selection

The component supports two category types:

- **Income**: For revenue and earning categories
- **Expense**: For spending and cost categories

Type selection is handled through:
1. Direct form control binding in templates
2. The `setType()` method for programmatic selection

## Form Validation

Both fields are required:

- **Name**: Must not be empty
- **Type**: Must be either 'Income' or 'Expense'

The form prevents submission when invalid:

```typescript
if (this.categoryForm.invalid) return;
```
