## Get Started in 5 Minutes

This quickstart guide will help you set up and run MyFinance UI on your local machine. You'll have the application running and be able to start tracking your finances in just a few steps.

<Note>
  Before you begin, make sure you have **Node.js 20+** and **pnpm** installed on your system.
</Note>

<Steps>
  <Step title="Clone the repository">
    First, clone the MyFinance UI repository to your local machine:
    
    ```bash
    git clone <repository-url>
    cd my-finance.ui
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required dependencies using pnpm:
    
    ```bash
    pnpm install
    ```
    
    This will install Angular 21, TailwindCSS, Highcharts, and all other dependencies defined in `package.json`.
  </Step>

  <Step title="Configure the API endpoint">
    Edit the environment configuration to point to your backend API:
    
    ```typescript src/environments/environment.ts
    export const env = {
        myFinanceApiUrl: 'http://localhost:5242/api'
    }
    ```
    
    <Warning>
      Make sure your backend API is running and accessible at this URL before proceeding. The application will not work without a backend connection.
    </Warning>
  </Step>

  <Step title="Start the development server">
    Launch the Angular development server:
    
    ```bash
    pnpm start
    ```
    
    Or using the Angular CLI directly:
    
    ```bash
    ng serve
    ```
    
    The application will be available at `http://localhost:4200/`
  </Step>

  <Step title="Log in to the application">
    Open your browser and navigate to `http://localhost:4200/`
    
    You'll be redirected to the login page at `/login`. Enter your credentials to access the application.
    
    <Info>
      The login credentials are managed by your backend API. Make sure you have a valid user account set up in your backend system.
    </Info>
  </Step>

  <Step title="Start tracking your finances">
    Once logged in, you'll be redirected to the transactions page. You can now:
    
    - **Create transactions**: Track your income and expenses
    - **Manage categories**: Organize transactions with custom categories
    - **View reports**: Analyze your financial data with monthly reports
    - **Visualize data**: See your spending patterns with interactive charts
  </Step>
</Steps>

## Quick Navigation

After logging in, the application provides several routes protected by authentication:

| Route | Description |
|-------|-------------|
| `/transactions` | Create and manage income/expense transactions |
| `/categories` | Create and organize transaction categories |
| `/reports` | View monthly financial reports and summaries |
| `/chart` | Visualize your financial data with interactive charts |

<Tip>
  The application uses route guards to protect these pages. If your JWT token expires, you'll be automatically redirected to the login page.
</Tip>

## Authentication Flow

MyFinance UI uses JWT (JSON Web Token) authentication:

1. **Login**: User submits credentials via the login form
2. **Token Storage**: JWT token is stored in localStorage as `access-token`
3. **Auto-attach**: HTTP interceptor automatically attaches the token to API requests
4. **Token Validation**: Token expiration is checked on each protected route access
5. **Auto-logout**: Expired tokens trigger automatic logout and redirect to login

You can see the authentication implementation in `src/app/services/Auth-service.ts`:

```typescript
async login(username: string, password: string) : Promise<boolean> {
    const tokenResponse = await fetch(`${this.apiUrl}/login`, {
        method: 'POST',
        headers: {
            'Content-Type': 'application/json'
        },
        body: JSON.stringify({ username, password })
    });
    if (!tokenResponse.ok) {
        console.error('Login failed:', tokenResponse.statusText);
        return false;
    }
    const tokenData: AuthModel = await tokenResponse.json();
    localStorage.setItem('access-token', tokenData.token);
    return true;
}
```

## Troubleshooting

### Port 4200 is already in use

If port 4200 is occupied, specify a different port:

```bash
ng serve --port 4300
```

### API connection failed

Verify that:
- Your backend API is running
- The API URL in `environment.ts` is correct
- There are no CORS issues blocking requests

### Login fails immediately

Check:
- Backend API is accessible at the configured URL
- You're using valid credentials from your backend database
- Network requests in browser DevTools for error details
